## 6. Playbooks

### 6.1 Data Verification Playbook
To verify data quality:
1. **Sample:** Select 10% random sample (50 entries).
2. **Test:** Send test email (or use verification API).
3. **Flag:** Mark bounced emails as "invalid".
4. **Replace:** Find replacement companies for invalid entries.
5. **Log:** Record deliverability rate (target: >95%).

### 6.2 Customer Complaint Playbook
If customer complains about data quality:
1. **Apologize:** Acknowledge the issue.
2. **Investigate:** Check specific entries complained about.
3. **Refund/Replace:** Offer refund or fresh dataset.
4. **Log:** Record complaint in quality log.
5. **Improve:** Update verification process if systemic issue.
